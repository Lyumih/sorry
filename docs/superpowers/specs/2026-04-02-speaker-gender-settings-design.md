# Дизайн: настройка пола формулировок (извинился/извинилась и плейсхолдеры)

**Дата:** 2026-04-02  
**Статус:** согласовано (brainstorming)  
**Решение пользователя:** вариант **B** — согласовать и глагол **извинился/извинилась**, и плейсхолдеры с **(а)** там, где речь о **ведущем дневник** (авторе записи).

## Цель

Убрать скобочные формы **извинился(лась)** и **сделал(а)** / **понял(а)** там, где грамматически подлежащее — **пользователь**. Выбор **«Мужчина» / «Женщина»** (по умолчанию **мужчина**) хранится локально и влияет на отображаемый текст формы, карточек в ленте, построение `buildApologyNarrative` и примеры в справке.

## Ограничение: ветка «У меня»

В фразе **«У меня извинился(лась) [кто] за …»** глагол относится к **тому, кто извинялся** (содержимое поля «кто»), а не к полу ведущего дневник. Глобальная настройка **не может** корректно заменить одну форму глагола для всех записей.

**Правило продукта:**

| Место | Поведение |
|--------|-----------|
| Ветка **«Я»** | **Извинился** / **извинилась** и плейсхолдеры **сделал/сделала**, **понял/поняла** (и аналоги про действия/рефлексию **автора**) — строго от настройки. |
| Ветка **«У меня»**, глагол перед «кто» | Оставить **извинился(лась)** (скобочная форма). |
| Ветка **«У меня»**, поле причины | Плейсхолдер **«повод, одной фразой»** — без рода (нейтрально). |
| Ветка **«У меня»**, блок выводов | Это рефлексия **ведущего** → **понял/поняла** и т.п. — от настройки. |

## UI

1. В шапке приложения (док с переключателем «Дневник / Статистика») **перед** кнопкой справки — иконка **настроек** (например `SettingOutlined`).
2. По нажатию — **Modal** «Настройки» (достаточно для одного блока опций).
3. Содержимое: краткий заголовок секции (смысл: **как показывать формулировки о вас** в дневнике) и **Radio.Group**: **Мужчина** / **Женщина**; значение по умолчанию — **мужчина**.
4. Применение: сразу при смене радио (без отдельной кнопки «Сохранить»), плюс стандартное закрытие модалки.
5. Доступность: у кнопки открытия — `aria-label` вроде **«Настройки отображения формулировок»** (и при необходимости `title`).

## Данные

- Тип значения: **`male` | `female`** (в коде — единое именование, например `SpeakerGender`).
- **Персистентность:** `localStorage` (рекомендуемый ключ, например **`sorry.speakerGender`**), синхронизация с клиентским состоянием при старте и при изменении.
- **Не входит** в экспорт JSON записей (это не содержимое дневника; перенос на другое устройство настройки не гарантируется).

**Архитектура состояния (рекомендация):** отдельный небольшой **Zustand**-store для префов + чтение/запись `localStorage` (или `zustand/middleware` `persist` с тем же ключом). Альтернатива — только React Context; для единообразия с существующим store предпочтительнее Zustand.

## Согласование кода

1. **`ApologyForm.tsx`** — подписка на настройку: нарратив **«Я»**, плейсхолдеры и подписи, зависящие от рода автора; ветка **«У меня»** — по таблице выше.
2. **`apologyNarrative.ts`** — функция `buildApologyNarrative` принимает **пол** (или объект полей, включающий пол) и строит строки согласно тем же правилам; **привести ветку `said_to_me` к текущей формулировке UI** («У меня извинился(лась) … за …»), заменив устаревший текст «попросили прощения».
3. **`EntryList.tsx`** (и любые другие вызовы `buildApologyNarrative`) — передавать актуальное значение настройки из store.
4. **`HelpModal.tsx`** — текст примеров согласовать с настройкой **или** явно указать, что формулировки подстраиваются под выбор в настройках (без устаревших скобок там, где уже выбран пол).
5. Вынести **хелперы строк** (форма глагола для «Я», фрагменты плейсхолдеров), чтобы не дублировать логику между формой и нарративом.

## Тестирование

- Юнит-тесты на хелперы / `buildApologyNarrative`: комбинации `male`/`female` × `i_said`/`said_to_me` — ожидаемые подстроки (ключевые слова и скобки там, где зафиксировано).

`npm run lint`, `npm run build`; вручную: смена пола в настройках, обе ветки формы, лента записей, справка.
