**Статус:** частично устарело по размещению экспорта — см. **`2026-04-02-journal-minimal-form-wrap-design.md`** и PRD v1.6 (экспорт на «Статистике»). Правила `.cursor/rules` обновлены отдельно.

## 1. Контекст

Репозиторий **sorry** («Прости, Извини») — локальный дневник извинений: React 19, Vite, TypeScript, Ant Design, Zustand, IndexedDB. Продуктовая правда — `PRD.md`; статусы итераций агента — `tasks.json`.

## 2. Выявленные расхождения с PRD (до правок)

- Экспорт JSON был на экране «Статистика»; по PRD §9 и §6 п.8 резервная копия должна быть на «Дневнике».
- Компонент `DataWarning` не использовался; по PRD §7 нужно явное предупреждение о локальном хранении на экране дневника.
- Справка (`HelpModal`) описывала экспорт со статистики.
- Период «год» в UI есть, в MVP PRD указаны день / неделя / месяц — оставляем как необязательное расширение UX без изменения PRD в этой итерации (при желании позже синхронизировать PRD).

## 4. Правила Cursor (`.cursor/rules/`)

Два файла с `alwaysApply: true` (язык — русский):

1. **Домен проекта** — стек, ссылка на `PRD.md` и `tasks.json`, принципы (данные только локально, русский UI, не ломать `schemaVersion` экспорта).
2. **Workflow superpowers** — при старте/фичах использовать цепочку навыков: `using-superpowers`; для продуктовых фич — `brainstorming` → спецификация → `writing-plans` → реализация; перед заявлением о готовности — `verification-before-completion`; при сбоях — `systematic-debugging`; по уместности — `test-driven-development`. Обновлять `tasks.json` после итераций по §13 PRD.

Ограничение: правила короткие, одна тема на файл, без дублирования полного текста навыков.
